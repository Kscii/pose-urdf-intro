# 世界坐标系（world_frame）方案说明

URDF 的 `base_link` 只是运动学根节点，本身不代表任何外部参考系。
`end_config.yaml` 里的 `urdf.world_frame` / `urdf.world_to_base`
就是给 `base_link` 挂一个静态变换、指向有意义的外部参考系的地方
（目前所有构型都是 identity 占位，即 world = base_link）。

下面是三种求 `base_link -> world` 静态变换的候选方案。

## 方案1：URDF + 下肢静止假设（脚底/底盘接地点）

**原理**：假设采集期间下肢（腿或轮式底盘）关节不动、只有手臂运动，用 URDF FK
算出 `base_link` 到脚底板/底盘接地点的静态变换，作为世界系。

**依赖**：只需要关节角 + URDF，无外部硬件。

**局限**：
- 本质是把 `base_link` 重新锚定到一个更有物理意义的点，不是"消除漂移"——下肢真静止的话，`base_link` 本身也没有漂移。
- 依赖"下肢真没动"的假设是否成立，以及 URDF 尺寸精度。
- 双足需要约定选哪只脚/双脚均值；轮式底盘上这个方案基本等价于已有的 `footprint` 约定，没有额外收益。
- FK 无法独立判断 `base_link` 相对重力的倾斜，姿态只能假定 `base_link` 本身水平。

**现状**：已用 `tools/build_world_frame_demo.py` 跑通真实计算（KUAVO_4PRO_GRIPPER 的腿部关节是
`role: ignored`，不接 H5、全程 URDF 零位，所以这份数据里该变换是精确值而非估计）。

## 方案2：头部相机 + 标定二维码

**原理**：假设头部（或第三方相机）相对 `base_link` 固定，用相机内参 + 已知尺寸的方形标记
（建议 ArUco/AprilTag，几何精度优于普通二维码）求出标记在相机系下的位姿，再通过相机外参转到
`base_link` 系，取标记中心作为世界系原点。

**依赖**：相机内参、相机相对 `base_link` 的外参（URDF 光心要准）、标记检测算法、已知尺寸的桌面标记。

**局限**：
- 仓库里目前没有相机内参标定、外参、标记检测代码，需要新建。
- 标记必须在视野内才能标定，失去视野就失锚。
- 光心 frame 容易出现 90° 轴系错误（URDF mount 点 ≠ 真实光心）。
- 是"桌面/任务局部"世界系，除非标记摆放位置可重复，否则不是跨 episode 稳定的全局系。

**现状**：仅在 demo mcap 里用手填模拟值演示了连线机制，数值不代表真实精度。

## 方案3：外部红外基站 + effector 红外传感器（含 pika 变体）

**原理**：effector 贴红外标记，选一个基站作为世界系；标定好 `base_link` 相对基站的静态变换后，
传感器直接上报世界系下的 end pose。pika 变体：pika 是无臂手持夹爪，采集前把它放在桌面标定点，
把此刻 pika 相对基站的变换当作"虚拟工业臂 `base_link` 相对世界系"的标定，采集到的夹爪轨迹当作
该虚拟工业臂的 end pose 做 IK 反解关节角，用于跨本体重定向（思路上等同 UMI 论文的做法）。

**依赖**：外部红外基站硬件、effector 红外标记、`base_link` 相对基站的标定。

**局限**：
- 需要额外硬件，基站有视线遮挡/量程限制。
- `base_link` 是移动底盘时，标定后底盘一动标定就失效，除非底盘也贴标记做在线跟踪。
- pika 方案里 IK 存在多解/关节跳变风险（需 warm-start）；人手轨迹可能超出目标臂可达空间，需显式标记失败帧；标定点摆放的可重复性也会引入误差。

**现状**：仅在 demo mcap 里用手填模拟值演示了连线机制。天然契合现有
`raw_pose_frame` / `raw_action_end` 归一化机制，真要接入时改动成本最低。

## 对比

| | 方案1 脚底FK | 方案2 二维码 | 方案3 红外基站 |
|---|---|---|---|
| 额外硬件 | 无 | 相机（可能已有） | 需外部基站 + 标记 |
| 精度 | 依赖 URDF 与下肢静止假设 | 依赖标定与检测算法，中等 | 最高，外部计量 |
| 覆盖范围 | 仅限下肢真正静止的 episode | 标记须在视野内 | 基站量程/视线覆盖内均可 |
| 仓库现状 | 可直接用现有 URDFKinematics 实现 | 无标定/检测代码 | 无标定数据，但 schema 已预留接口 |
| demo 状态 | 真实计算 | 模拟数值 | 模拟数值 |
